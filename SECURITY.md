# Security Policy

## Supported Versions

This repository is actively maintained on the default branch.

Consumers of this template are encouraged to regularly incorporate updates from the latest version.

## Reporting a Vulnerability

If you discover a security vulnerability, do not disclose it publicly.

Instead:

1. Notify the project maintainers through approved internal channels.
2. Provide a detailed description of the issue.
3. Include steps to reproduce where possible.
4. Include any relevant logs, screenshots, or proof-of-concept material.

## Response Process

Reported vulnerabilities will be:

* Acknowledged as soon as practical.
* Assessed for severity and impact.
* Investigated and remediated where appropriate.
* Communicated to affected stakeholders when necessary.

## Dependency Management

Projects generated from this template should:

* Keep dependencies up to date.
* Regularly review dependency vulnerabilities.
* Apply security updates promptly.
* Minimize unnecessary dependencies.

## Responsible Disclosure

Please allow maintainers reasonable time to investigate and address reported vulnerabilities before sharing details more broadly.
